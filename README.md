# Artzen (artzen.pk)

Premium Islamic wall art, MDF wood calligraphy, and home decor. This storefront is shipped as a static Next.js export for CSR-friendly hosting.

## Stack

- **Next.js 16** (App Router) with static export (`output: "export"`)
- **React 19** + **Tailwind CSS**
- Static hosting friendly (`out/` output, shared hosting rewrite support in `public/.htaccess`)

## Development

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Production build (static export)

```bash
npm run build
```

Build output is generated for static hosting. Use your host upload/deploy flow for the exported site.

## Hosting notes

- `next.config.ts` is configured for static export and image compatibility on static hosts.
- `public/.htaccess` supports extensionless URLs and static 404 fallback on Apache/shared hosting.
- `public/_headers`, `vercel.json`, and `netlify.toml` provide header examples for common hosts.

## Orders

Checkout submissions are handled client-side and can be sent to Formspree.

Required env var:

```bash
NEXT_PUBLIC_FORMSPREE_ID=your_form_id
```

Optional env vars:

- `NEXT_PUBLIC_SITE_URL` (absolute links in order payloads)
- `NEXT_PUBLIC_WHATSAPP_PHONE` (WhatsApp fallback button/message)

To switch provider, edit `src/app/checkout/CheckoutForm.tsx`.

## SEO and prerender

- `sitemap.xml` is generated by `src/app/sitemap.ts`
- `robots.txt` is generated by `src/app/robots.ts`
- Global metadata and JSON-LD schema are defined in `src/app/layout.tsx`
- Product and collection routes use static params/metadata for prerendered SEO pages
- Because the site is a static export, SEO surfaces update on rebuild/redeploy (not at request time)

## Project structure

- `src/app/` — app routes and page-level SEO metadata
- `src/components/` — storefront UI components
- `src/context/` — client state (cart, favorites, etc.)
- `src/lib/data.ts` and `src/lib/data.generated.ts` — product/collection catalog
